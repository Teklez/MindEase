# RAG Phase 0 + Phase 1 — Standalone Implementation Plan

## Overview

This sub-plan implements **Phase 0** (pgvector + `memory_chunks` + `/embed` endpoint + `MemoryService`) and **Phase 1** (text-chat retrieval) from the parent RAG plan, as a single ready-to-ship unit. Phase 0 puts the plumbing in place with zero call-site changes (no behavioral regression possible). Phase 1 rewrites the one place where the chat prompt is built — `ChatService.process_message_stream` — to layer in cross-conversation memory, recent mood, latest assessment severities, and retrieved chunks. Both phases ship together because Phase 0 alone has no user-visible effect and would otherwise sit as dead infrastructure.

Decisions locked for this sub-plan (confirmed before drafting):

- **Tests**: no automated test suite exists in the project (no `pytest`, no `mypy`, no CI). This sub-plan adds **smoke tests only** — shell commands and one-off scripts to verify behavior. Building a real test harness is out of scope and deferred.
- **JSONB column name**: `attrs` (not `metadata`, which collides with SQLAlchemy's `Base.metadata`).
- **Indexing timing**: embed the user content **exactly once** per turn, in parallel with the crisis check; reuse that embedding for both retrieval and the user-message chunk insert. AI-message indexing is fire-and-forget after the stream completes.
- **Backfill**: manual only — `python -m app.seeds.backfill_memory`. No auto-on-boot.

## Desired End State

After Phase 0 + Phase 1 ship:

1. `docker compose up` brings up `pgvector/pgvector:pg16` on the same volume, with `vector` and `pgcrypto` extensions enabled. The `memory_chunks` table exists with cosine + filter indexes.
2. The ai-service exposes `POST /embed` returning 768-d vectors from Gemini `text-embedding-004`, batching up to 100 inputs per call. The backend's `AIClient.embed([...])` proxies to it.
3. A `MemoryService` (`backend/app/services/memory_service.py`) is the single API for indexing and retrieving chunks, with a precomputed-vector escape hatch (`query_vec=`) so callers that already embedded the query don't pay for a second round-trip.
4. `ChatService.process_message_stream` builds a layered system prompt:
   ```
   <static SYSTEM_PROMPT>
   ## About this user           (display name, mood snapshot, latest screenings)
   ## Recent mood               (last 7 days)
   ## Latest screenings         (latest UserAssessment per type)
   ## Relevant past moments     (top-k retrieved chunks from other conversations)
   ```
   followed by a last-10-message rolling window of the current conversation. The user-content embedding is computed once and reused for retrieval and the user-message chunk insert. The AI-message chunk is embedded and inserted in a fire-and-forget task after the stream ends.
5. `MoodEntry.create_entry` indexes notes (`source_kind="mood_note"`) and `AssessmentService.submit` indexes `feedback_text` (`source_kind="assessment_result"`) on creation, so future turns retrieve them.
6. A one-shot backfill script (`python -m app.seeds.backfill_memory`) embeds existing `messages.content`, `mood_entries.note`, and `user_assessments.feedback_text` into `memory_chunks`, idempotent via a partial unique index on `(source_kind, source_id)`.

## Key Discoveries from Re-verification

Things that drifted or didn't match the parent plan, which this sub-plan corrects:

- The mood-stats method is **`get_stats`**, not `get_user_stats`.
- `AssessmentService.submit` is **lines 114-182** of `assessment_service.py`, not the broader range the parent plan listed.
- AI-message persist starts at **line 197** (line 195 is the `"".join(full_response)` step).
- There is **no shared `httpx.AsyncClient`** on `AIClient` — every method opens a new one per call. `embed()` will follow the same pattern (no refactor).
- `metadata` is a reserved attribute on SQLAlchemy's `DeclarativeBase`. We rename the JSONB column to **`attrs`** everywhere in this plan.
- `pgcrypto` is depended on for `gen_random_uuid()` but never `CREATE EXTENSION`-ed. The new migration enables it defensively.
- `alembic/env.py:18` only imports three models. We add `MemoryChunk` to that import list so `--autogenerate` ever runs against it cleanly.
- `Conversation.status` (not `deleted_at`) is how soft-delete works. The backfill and retrieval filter against `status != 'archived'` for `message` chunks.
- `get_db` auto-commits at request end, but `ChatService` does not use `get_db` for its writes — it manages commits explicitly. `MemoryService.index` will use `db.flush()` for the inline user-message chunk insert and let the existing surrounding `db.commit()` boundary in `process_message_stream` carry it. For background AI-message indexing, the fire-and-forget task opens its own session via `async_session_maker()` and commits there.

## What We're NOT Doing

- No automated test framework. Smoke checks only.
- No Phase 2+ scope. Resources stay on the keyword recommender, voice stays browser-direct, group AI stays on its 10-msg window, crisis stays on substrings.
- No `profile_fact` extraction (Phase 7), no conversation summarization (Phase 6). The retrieval `kinds` filter will *allow* those source-kinds so a later phase can add them without re-touching ChatService — but no producer code lands here.
- No Memory-UI Settings page (Phase 7). The `/api/v1/me/memory` REST endpoint and `/settings/memory` are deferred. Users cannot view or delete chunks in v1 of this sub-plan; deletion is via direct SQL only.
- No global / `user_id IS NULL` chunks. The `user_id` column stays `NOT NULL` in this sub-plan; Phase 2 (resources) and Phase 5 (crisis phrases) will add the `DROP NOT NULL` + check constraint when they land.
- No backend `httpx` connection-pool refactor.
- No change to ai-service's `/generate`, `/check-crisis`, or `/translate` routes. Only `/embed` is added.
- No change to the Amharic translation pipeline. Embeddings are computed on the raw text (Gemini `text-embedding-004` is multilingual).
- No frontend changes. Phase 1 is entirely backend.

## Implementation Approach

Two phases, both backend-only, shipping together in one branch:

- **Phase 0** is non-behavioral: image swap, one Alembic migration, new model, new ai-service route, new client method, new `MemoryService`. Zero existing call sites change. If we ship Phase 0 alone, nothing in the UI changes.
- **Phase 1** is the prompt-builder rewrite plus three indexing call sites (chat-user-msg, chat-ai-msg, mood note, assessment feedback) plus a backfill script.

Both can be reverted independently: Phase 1 by reverting `chat_service.py` and the two service changes; Phase 0 by reverting the migration (drops table) and the image.

#### 5. ai-service: `/embed` route

**File**: `ai-service/app/services/embedder.py` *(new)*

Mirrors the `TranslationService` pattern from `ai-service/app/services/translator.py:105-111`: module-level singleton, lazy Gemini client, async call.

```python
from __future__ import annotations
import logging
from google import genai
from google.genai import types

from app.config import get_settings

logger = logging.getLogger(__name__)


class EmbedderService:
    def __init__(self) -> None:
        settings = get_settings()
        self._model = settings.EMBED_MODEL_NAME
        self._dim = settings.EMBED_DIM
        self._client = genai.Client(api_key=settings.GEMINI_API_KEY) if settings.GEMINI_API_KEY else None

    async def embed_batch(self, texts: list[str]) -> list[list[float]]:
        if not texts:
            return []
        if self._client is None:
            raise RuntimeError("GEMINI_API_KEY is not configured")
        # Gemini supports up to 100 inputs per call for text-embedding-004.
        out: list[list[float]] = []
        for i in range(0, len(texts), 100):
            batch = texts[i : i + 100]
            resp = await self._client.aio.models.embed_content(
                model=self._model,
                contents=batch,
                config=types.EmbedContentConfig(
                    task_type="RETRIEVAL_DOCUMENT",
                    output_dimensionality=self._dim,
                ),
            )
            out.extend([e.values for e in resp.embeddings])
        return out


embedder = EmbedderService()
```

> Note: the actual `task_type` enum surface in `google-genai` is `EmbedContentConfig(task_type=…)`. If a future SDK bump renames the field, this is the only place to change. `RETRIEVAL_DOCUMENT` is correct for stored chunks; the backend will pass the same task type for both index and query in v1 (simplification — Gemini accepts mismatched task types; using both consistently keeps numbers stable).

```python
from fastapi import APIRouter, HTTPException

from app.services.embedder import embedder

router = APIRouter()


@router.post("")
async def embed(body: dict) -> dict:
    """POST /embed — body: {"texts": [str, ...]} → {"embeddings": [[float; 768], ...]}."""
    texts = body.get("texts") or []
    if not isinstance(texts, list) or not all(isinstance(t, str) for t in texts):
        raise HTTPException(status_code=400, detail="`texts` must be a list of strings")
    if not texts:
        return {"embeddings": []}
    try:
        vectors = await embedder.embed_batch(texts)
    except Exception as exc:  # log and surface a 502 so the backend can fall back gracefully
        raise HTTPException(status_code=502, detail=f"embed failed: {exc}") from exc
    return {"embeddings": vectors}
```

```python
        """POST to {base_url}/embed
        Body: {"texts": [str, ...]} → returns list of 768-d float vectors.
        Raises on HTTP error so callers can catch and degrade gracefully.
```

#### 7. `MemoryService`

**File**: `backend/app/services/memory_service.py` *(new)*

The retrieval API takes either `query_text` (embed inline) or `query_vec` (precomputed). When both are present, `query_vec` wins. Phase 1 uses `query_vec` to avoid a second `/embed` round-trip per turn.

```python
from __future__ import annotations
import logging
import uuid
from dataclasses import dataclass
from datetime import datetime
from typing import Any, Sequence

from sqlalchemy import delete, select
from sqlalchemy.ext.asyncio import AsyncSession

from app.models.memory_chunk import MemoryChunk
from app.services.ai_client import AIClient

logger = logging.getLogger(__name__)


@dataclass
class IndexItem:
    user_id: uuid.UUID
    source_kind: str
    text: str
    source_id: uuid.UUID | None = None
    conversation_id: uuid.UUID | None = None
    group_id: uuid.UUID | None = None
    attrs: dict[str, Any] | None = None
    embedding: list[float] | None = None  # if set, no embed call is made


class MemoryService:
    def __init__(self) -> None:
        self.ai_client = AIClient()

    async def index(
        self,
        db: AsyncSession,
        *,
        user_id: uuid.UUID,
        source_kind: str,
        text: str,
        source_id: uuid.UUID | None = None,
        conversation_id: uuid.UUID | None = None,
        group_id: uuid.UUID | None = None,
        attrs: dict[str, Any] | None = None,
        embedding: list[float] | None = None,
    ) -> MemoryChunk:
        """Index a single chunk. If `embedding` is provided, no /embed call is made.
        Caller is responsible for the surrounding commit boundary.
        On the unique (source_kind, source_id) collision, returns the existing row.
        """
        if not text or not text.strip():
            raise ValueError("text must be non-empty")
        if embedding is None:
            vectors = await self.ai_client.embed([text])
            if not vectors:
                raise RuntimeError("embed returned no vectors")
            embedding = vectors[0]

        chunk = MemoryChunk(
            user_id=user_id,
            source_kind=source_kind,
            source_id=source_id,
            conversation_id=conversation_id,
            group_id=group_id,
            text=text,
            embedding=embedding,
            attrs=attrs,
        )
        db.add(chunk)
        try:
            await db.flush()
        except Exception:
            # Most likely cause: unique-index collision on (source_kind, source_id).
            # Roll back this savepoint, fetch the existing row, return it.
            await db.rollback()
            if source_id is not None:
                result = await db.execute(
                    select(MemoryChunk).where(
                        MemoryChunk.source_kind == source_kind,
                        MemoryChunk.source_id == source_id,
                    )
                )
                existing = result.scalar_one_or_none()
                if existing is not None:
                    return existing
            raise
        return chunk

    async def index_many(
        self, db: AsyncSession, items: Sequence[IndexItem],
    ) -> int:
        """Batch-embed and insert N items in two round-trips (one /embed call + bulk insert).
        Used by the backfill script. Returns the number of rows actually inserted
        (collisions on (source_kind, source_id) are silently skipped).
        """
        items = list(items)
        if not items:
            return 0
        to_embed = [it for it in items if it.embedding is None]
        if to_embed:
            vectors = await self.ai_client.embed([it.text for it in to_embed])
            if len(vectors) != len(to_embed):
                raise RuntimeError(
                    f"embed returned {len(vectors)} vectors for {len(to_embed)} texts"
                )
            for it, vec in zip(to_embed, vectors):
                it.embedding = vec

        # Use one INSERT per row inside a savepoint so a single collision doesn't
        # abort the whole batch. asyncpg + SQLAlchemy doesn't expose ON CONFLICT
        # cleanly without raw SQL; this loop is simple and the backfill is one-shot.
        inserted = 0
        for it in items:
            try:
                await self.index(
                    db,
                    user_id=it.user_id,
                    source_kind=it.source_kind,
                    text=it.text,
                    source_id=it.source_id,
                    conversation_id=it.conversation_id,
                    group_id=it.group_id,
                    attrs=it.attrs,
                    embedding=it.embedding,
                )
                inserted += 1
            except Exception as exc:
                logger.warning(
                    "skip chunk source_kind=%s source_id=%s: %s",
                    it.source_kind, it.source_id, exc,
                )
        return inserted

    async def retrieve(
        self,
        db: AsyncSession,
        *,
        user_id: uuid.UUID,
        query_text: str | None = None,
        query_vec: list[float] | None = None,
        k: int = 6,
        kinds: list[str] | None = None,
        exclude_conversation_id: uuid.UUID | None = None,
        since: datetime | None = None,
    ) -> list[MemoryChunk]:
        """Cosine-similarity top-k retrieval scoped to `user_id`.

        Provide either `query_text` (will be embedded inline) or `query_vec`
        (precomputed — preferred when the caller already embedded the text).
        """
        if query_vec is None:
            if not query_text:
                return []
            vectors = await self.ai_client.embed([query_text])
            if not vectors:
                return []
            query_vec = vectors[0]

        stmt = select(MemoryChunk).where(MemoryChunk.user_id == user_id)
        if kinds:
            stmt = stmt.where(MemoryChunk.source_kind.in_(kinds))
        if exclude_conversation_id is not None:
            stmt = stmt.where(
                (MemoryChunk.conversation_id != exclude_conversation_id)
                | (MemoryChunk.conversation_id.is_(None))
            )
        if since is not None:
            stmt = stmt.where(MemoryChunk.created_at >= since)
        stmt = stmt.order_by(MemoryChunk.embedding.cosine_distance(query_vec)).limit(k)

        result = await db.execute(stmt)
        return list(result.scalars().all())


memory_service = MemoryService()
```

```python
def _format_chunks(chunks: list) -> str:
    """Render retrieved MemoryChunks as `[YYYY-MM-DD, kind] text` lines so the LLM
    can cite them by time."""
    lines: list[str] = []
    for c in chunks:
        d = c.created_at.date().isoformat()
        # Trim to keep the system prompt bounded — 500 chars per chunk.
        text = (c.text or "").strip().replace("\n", " ")
        if len(text) > 500:

        lines.append(f"[{d}, {c.source_kind}] {text}")
```

```python
async def _backfill_messages(db) -> int:
    """Embed and index every existing non-empty Message from non-archived conversations."""
    total = 0
    # Stream by ascending timestamp so ordering is stable.
    result = await db.execute(
        select(Message, Conversation.user_id)

        .where(Conversation.status != "archived")
```

## Performance Considerations

- **Added latency per chat turn**: one new round-trip to `/embed` for the user content. Mitigated by running it in parallel with the existing `/check-crisis` call via `asyncio.gather` — the new floor is `max(embed_latency, crisis_latency)`, not their sum. Gemini `text-embedding-004` typically returns in 100-300 ms.
- **Retrieval query**: one `SELECT ... ORDER BY embedding <=> $1 LIMIT 6` against the ivfflat index. Cheap (< 20 ms) up to ~100k chunks. The ivfflat `lists=100` is the standard rule-of-thumb start; revisit with `SET LOCAL ivfflat.probes` tuning if recall drops noticeably.
- **AI-message indexing is fire-and-forget**: never blocks the `done` event. Worst case if the task crashes mid-flight is one missing chunk for the next turn's retrieval, recoverable by running the backfill.
- **User-message indexing is inline** (re-using the already-computed vector). Cost is one INSERT (~5 ms). Justified by guaranteeing the chunk is queryable on the very next turn — important for short back-and-forth chats.
- **Memory budget**: 6 retrieved chunks × ≤ 500 chars + profile block (≤ 1 KB) + last-10 messages stays well under any plausible LLM context window.
- **Backfill is one-shot**: a moderate prod dataset (say 50k messages) → 500 `/embed` calls of 100 each → ≈ 2-3 minutes wall-clock. Safe to run during low traffic.

## Migration / Rollback Notes

- **Forward migration** is the one new Alembic revision (`add_memory_chunks`). It is forward-only in spirit but writes a complete `downgrade()` that drops the table and indexes (leaves `vector` and `pgcrypto` extensions installed — cheap, harmless).
- **Image swap**: `pgvector/pgvector:pg16` is a drop-in for `postgres:16-alpine` on the same `postgres_data` volume. To roll back: change the image back to `postgres:16-alpine`, then run `alembic downgrade -1`. The `vector` extension's presence in pg-catalog does not break the alpine image (the extension files just won't be there; nothing tries to load them after the table is dropped).
- **No data destruction**. Existing tables are untouched. The new `memory_chunks` table is additive only.
- **Cost**: Gemini `text-embedding-004` is on Google's free tier up to standard quota; a steady-state of < 100 embed calls / min stays free. Worth wiring usage alerts but no architectural impact.
